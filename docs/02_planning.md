# 🗂 超軽量PM「計画（スケジュール）」テンプレ

※ 立ち上げテンプレの 次にそのまま使う想定

# 1️⃣ やること全部出す（順番・日付なし）

まず、何も考えずに吐き出します。

・〇〇を調べる
・〇〇を触る
・〇〇の資料を見る
・〇〇を設定する
・メモを書く


- 👉 この段階では日付を入れない
- 👉 これはWBS未満、ただの棚卸し

# 2️⃣ 粒度をそろえる（ここがPM）

次に、各項目を見て自問します。

「これ、1回の着席で終わる？」

❌ ダメな例
・G検定を勉強する

✅ OKな例
・G検定の目次を見る
・AI倫理の章を1回読む


👉 1項目＝10〜30分 が目安
👉 これで「現実と衝突しにくくなる」

# 3️⃣ 日付を入れる（雑でいい）

ここで初めて日付。

10/10（火）
・目次を見る
・用語を5つ眺める

10/12（木）
・AI倫理の章を1回読む

重要ルール

空白日があってOK

毎日埋めない

週に2〜3日で十分

👉 詰めないことがPM

# 4️⃣ 余白日（バッファ）を先に作る

最初からこれを書きます。

【予備】
・体調が悪ければ何もしない
・気が向いたら軽い項目だけ


👉 これは甘えではなく
👉 リスク管理

# 5️⃣ 「やらなくていい日」の定義

計画に、これを入れてください。

今日はこれができたら成功：
・1項目できたらOK


👉 ゼロじゃなければ前進

# 6️⃣ 週1回の計画見直し（5分）

この3行だけ書く：

・実際にできたこと
・重かった項目
・来週削るもの


👉 追加しない、削る

## 📌 この計画テンプレの正体

PMBOKで言うと：

| PMBOK用語  | やっていること  |
| -------- | -------- |
| スケジュール管理 | 粒度をそろえる  |
| リスク管理    | バッファを入れる |
| 統合管理     | 削って調整する  |
| 品質管理     | 続いたかどうか  |
